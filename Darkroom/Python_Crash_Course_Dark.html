1. Variables & Data Types

📂 Dataset used in this lesson: photographers.csv

Think of a Variable as the label on a film canister. It tells Python what is inside.

# Defining our first photo data variables photo_name = "World_Cup_Final_001.jpg" shutter_speed = "1/1000" iso = 800 camera_body = "Canon R5" # Print them out print(photo_name) print("ISO was:", iso)

Python stores three main types of data you will use constantly:

# Text (called a string) photographer = "Satdiva" # Number (called an integer) total_shots = 750000 # True / False (called a boolean) is_raw = True
When you are foundationed, all becomes obvious. Variables are the foundation. Everything else in Python builds on this.

2. Lists & Dictionaries

📂 Dataset used in this lesson: photographers.csv · shoots.csv

A List is like a contact sheet. It holds many items in one place in a specific order.

equestrian_events = ["Show Jumping", "Dressage", "Polo"] # Get the first item (Python counts from 0) print(equestrian_events[0]) # → "Show Jumping" print(equestrian_events[2]) # → "Polo" # How many items? print(len(equestrian_events)) # → 3

A Dictionary is like a camera bag. Each item has a name (key) and a value.

photographer = { "name": "Satdiva", "credential": "USA06591", "speciality": "Equestrian", "total_shoots": 4200 } # Access by key name print(photographer["name"]) # → "Satdiva" print(photographer["speciality"]) # → "Equestrian"
Think of it this way: A List is ordered like a film roll. A Dictionary is labelled like a file cabinet.

3. Loops & Logic

📂 Dataset used in this lesson: shoots.csv

A Loop lets Python repeat a task automatically — like processing every photo in a folder without doing it one by one.

shots = ["IMG_001.jpg", "IMG_002.jpg", "IMG_003.jpg"] for shot in shots: print("Processing:", shot) # Output: # Processing: IMG_001.jpg # Processing: IMG_002.jpg # Processing: IMG_003.jpg

Logic (if / else) lets Python make decisions based on data.

iso = 3200 if iso > 1600: print("High ISO — check for noise") elif iso > 800: print("Medium ISO — acceptable") else: print("Clean ISO — excellent quality")
Combined power: Loop through every shoot. For each one, check the ISO. Flag the noisy ones automatically.

4. Functions

📂 Dataset used in this lesson: shoots.csv · equipment.csv

A Function is a reusable tool you build once and use forever. Like a lens — you buy it once, you use it on any camera body.

# Define the function once def describe_shoot(event, location, shots_taken): print(f"Event: {event}") print(f"Location: {location}") print(f"Total shots: {shots_taken}") # Use it as many times as you need describe_shoot("Wellington Polo", "Florida", 840) describe_shoot("Grand Prix Jumping", "Geneva", 1200)

Functions can also return a value — like a calculation result.

def keeper_rate(total, keepers): rate = (keepers / total) * 100 return round(rate, 1) result = keeper_rate(840, 92) print(f"Keeper rate: {result}%") # → Keeper rate: 11.0%
Rule of thumb: If you write the same code twice, make it a function.

5. Reading CSV Files

📂 Dataset used in this lesson: photographers.csv

CSV stands for Comma-Separated Values. Your archive data lives in CSV files. This lesson shows you how to open and read them with Python.

import csv # Open the photographers dataset with open("photographers.csv", "r") as file: reader = csv.DictReader(file) for row in reader: print(row["name"], "—", row["speciality"])

photographers.csv structure

id, name, credential, speciality, country 1, Satdiva, USA06591, Equestrian, USA 2, David, USA00234, Polo, USA 3, Snoopy, USA00089, Equestrian, USA
DictReader reads each row as a dictionary — so you access columns by name, not by position. Much easier to read.

6. Pandas DataFrames

📂 Dataset used in this lesson: photographers.csv · shoots.csv

Pandas is a Python library that turns your CSV into a powerful table you can query like a database.

import pandas as pd # Load the dataset df = pd.read_csv("shoots.csv") # See the first 5 rows print(df.head()) # How many rows and columns? print(df.shape) # → (750000, 8) # Column names print(df.columns)

shoots.csv structure

id, photographer_id, event, location, year, shots, keepers 1, 1, Wellington Polo, Florida, 2019, 840, 92 2, 1, Grand Prix, Geneva, 2022, 1200, 140 3, 3, Dressage Cup, Aachen, 2021, 600, 78
750,000 rows — that is your full archive. Pandas handles it instantly. Python without Pandas would take minutes.

7. Filtering Shoots

📂 Dataset used in this lesson: shoots.csv

Filtering lets you ask questions of your data. Which shoots were in Florida? Which year had the most shots? Find them in one line.

import pandas as pd df = pd.read_csv("shoots.csv") # All shoots in Florida florida = df[df["location"] == "Florida"] print(florida) # All shoots after 2020 recent = df[df["year"] > 2020] print(recent.head()) # Shoots with more than 1000 shots taken heavy = df[df["shots"] > 1000] print(f"Heavy shoots: {len(heavy)}")

Combine filters using & (and) or | (or):

# Florida AND after 2020 filtered = df[(df["location"] == "Florida") & (df["year"] > 2020)] print(filtered)
This is the power. 750,000 records. One line of code. The answer is instant.

8. Visualisation

📂 Dataset used in this lesson: shoots.csv · photographers.csv

Matplotlib turns your data into charts. See your archive at a glance — shoots per year, locations, keeper rates.

import pandas as pd import matplotlib.pyplot as plt df = pd.read_csv("shoots.csv") # Count shoots per year by_year = df.groupby("year")["id"].count() # Plot it by_year.plot(kind="bar", color="#6bcf8a") plt.title("Shoots Per Year") plt.xlabel("Year") plt.ylabel("Number of Shoots") plt.tight_layout() plt.show()
You have completed the foundation. Variables → Lists → Loops → Functions → CSV → Pandas → Filter → Visualise. That is the full pipeline. Your archive is now programmable.
