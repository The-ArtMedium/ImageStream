What is SQL?

Understanding databases before writing a single line

SQL stands for Structured Query Language. It is the language you use to talk to a database — to ask questions, retrieve data, add new records, or make changes.

Our database is called photography_db. It contains photographers from 20 countries, their shoots around the world, and the equipment they own. This is the same data used in the Python course — two languages, one dataset.

Key Terms

A database is a collection of organised data. A table is one set of related data. A row is one record. A column is one piece of information about that record.

Convention

SQL keywords like SELECT and FROM are not case-sensitive — but we write them in CAPITALS so they stand out clearly from table and column names.

SELECT — Reading Data

Your first SQL query. Ask the database for data.

The most important SQL command is SELECT. You will use it in almost every query you ever write.

Syntax
SELECT column1, column2 FROM table_name;

Get all columns

SQL
SELECT * FROM photographers;

Get specific columns

SQL
SELECT name, country, specialty, rating
FROM photographers;

Always end your SQL statement with a semicolon ; — it tells the database where the statement ends.

WHERE — Filtering Rows

Ask for only the rows that match a condition

SELECT gives you all rows. WHERE filters — only give me rows where this condition is true.

Syntax
SELECT columns FROM table WHERE condition;

Find all Portrait photographers

SQL
SELECT name, country, rating
FROM photographers
WHERE specialty = 'Portrait';

Photographers rated above 4.5

SQL
SELECT name, country, rating
FROM photographers
WHERE rating > 4.5;
Result

Combining conditions

SQL
-- Wildlife photographers with more than 10 years experience
SELECT name, country, years_experience
FROM photographers
WHERE specialty = 'Wildlife'
  AND years_experience > 10;

ORDER BY — Sorting Results

Control the order results come back in

ORDER BY lets you sort by any column — rating, year, name, fee, or anything else.

Syntax
SELECT columns FROM table ORDER BY column ASC|DESC;
SQL
SELECT name, country, rating
FROM photographers
ORDER BY rating DESC;
Result (top 4)
namecountryrating
Sarah OduyaKenya4.8
Mei LinChina4.7
Priya NairIndia4.6
Nour KhalilLebanon4.5
ASC vs DESC

ASC = Ascending — lowest to highest, A to Z. This is the default.
DESC = Descending — highest to lowest, Z to A.

SQL
-- Specialty A-Z, then by rating highest first
SELECT name, specialty, rating
FROM photographers
ORDER BY specialty ASC, rating DESC;

LIMIT & DISTINCT

Get only the top results and remove duplicates

LIMIT — Top N results

SQL
-- Top 5 highest rated photographers
SELECT name, country, rating
FROM photographers
ORDER BY rating DESC
LIMIT 5;

DISTINCT — Remove duplicates

SQL
-- What specialties exist?
SELECT DISTINCT specialty
FROM photographers;
Result
specialty
Wildlife
Street
Portrait
Documentary
Landscape
Common Pattern

ORDER BY + LIMIT = give me the top N. One of the most used patterns in SQL. Top 10 photographers. Highest paying shoots. Most recent 5 jobs.

Aggregates

COUNT, AVG, MIN, MAX

Summarise many rows into one meaningful number

Aggregate functions crunch many rows into a single result. How many photographers? What is the average rating? What is the highest fee ever charged?

COUNT

SQL
SELECT COUNT(*) AS total_photographers
FROM photographers;
Result
total_photographers
20

AVG, MIN, MAX, SUM on shoots

SQL
SELECT
  AVG(fee_usd) AS avg_fee,
  MIN(fee_usd) AS lowest_fee,
  MAX(fee_usd) AS highest_fee,
  SUM(fee_usd) AS total_earned
FROM shoots;
Result
avg_feelowest_feehighest_feetotal_earned
572.400300059290
AS — Renaming columns

The AS keyword gives a column a readable name in results. Without it you see AVG(fee_usd). With it you see avg_fee. Always use AS to make results clear.

07
Aggregates

GROUP BY & HAVING

Group rows and filter the groups themselves

GROUP BY calculates aggregate values for each group — average rating per specialty, total fees per year, count of shoots per location.

SQL
SELECT specialty, AVG(rating) AS avg_rating
FROM photographers
GROUP BY specialty
ORDER BY avg_rating DESC;
Result
specialtyavg_rating
Documentary4.45
Portrait4.40
Landscape4.35
Wildlife4.20
Street3.90

HAVING — Filter the groups

SQL
-- Locations with more than 5 shoots
SELECT location, COUNT(*) AS shoot_count
FROM shoots
GROUP BY location
HAVING COUNT(*) > 5
ORDER BY shoot_count DESC;
WHERE vs HAVING

Use WHERE to filter rows before grouping. Use HAVING to filter groups after grouping. You cannot use COUNT() inside WHERE — that is what HAVING is for.

Relationships

JOIN — Connecting Two Tables

Combine photographers with their shoots

The shoots table has a photographer_id column that links each shoot to the photographer who did it. JOIN reunites them.

Why separate tables?

If we stored the photographer's name in every shoot row, we'd have to update many rows every time a name changed. With separate tables, the name is stored once. JOIN brings them together when needed.

SQL
SELECT
  p.name,
  p.country,
  s.location,
  s.fee_usd,
  s.year
FROM photographers p
JOIN shoots s
  ON p.photographer_id = s.photographer_id;
Result (sample)
namecountrylocationfee_usdyear
Amara DialloSenegalKyiv3502024
Amara DialloSenegalKolkata7502023
Priya NairIndiaLagos15002022
TypeReturnsUse when
INNER JOINOnly rows that match in both tablesYou only want complete matches
LEFT JOINAll rows from left table, matched or notAll photographers even if no shoots
Aliases

We wrote photographers p and shoots s. These short letters save typing. Instead of photographers.name we write p.name. Always use aliases when joining tables.

Relationships

Multiple JOINs

Photographers + shoots + equipment in one query

You can chain multiple JOINs. Each one adds another table to your result.

SQL
SELECT
  p.name,
  s.location,
  s.fee_usd,
  e.brand,
  e.model
FROM photographers p
JOIN shoots s ON p.photographer_id = s.photographer_id
JOIN equipment e ON p.photographer_id = e.photographer_id
WHERE p.name = 'Priya Nair';

Top earners — JOIN + GROUP BY

SQL
SELECT
  p.name,
  p.rating,
  SUM(s.fee_usd) AS total_earned,
  COUNT(s.shoot_id) AS total_shoots
FROM photographers p
JOIN shoots s ON p.photographer_id = s.photographer_id
GROUP BY p.photographer_id, p.name, p.rating
ORDER BY total_earned DESC
LIMIT 5;

Subqueries

A query inside a query

A subquery is a SELECT inside another SELECT. The inner query runs first. The outer query uses its result.

Photographers rated above the average

SQL
SELECT name, country, rating
FROM photographers
WHERE rating > (
  SELECT AVG(rating) FROM photographers
)
ORDER BY rating DESC;
How to read a subquery

Read the inner query first (inside the parentheses). It returns one number — the average. The outer query uses that number in its WHERE clause.

Photographers who have shot in Tokyo

SQL
SELECT name, country, specialty
FROM photographers
WHERE photographer_id IN (
  SELECT photographer_id
  FROM shoots
  WHERE location = 'Tokyo'
);

Advanced

INSERT, UPDATE & DELETE

Add, change and remove data — safely

So far we have only read data. These three commands let you change data. Learn them carefully.

INSERT — Add a new photographer

SQL
INSERT INTO photographers
  (name, country, specialty, years_experience, rating)
VALUES
  ('Ana Lima', 'Brazil', 'Documentary', 8, 4.3);

UPDATE — Change existing data

SQL
UPDATE photographers
SET rating = 4.9
WHERE photographer_id = 3;

DELETE — Remove a row

SQL
DELETE FROM shoots
WHERE shoot_id = 15;
Golden Rule

ALWAYS include a WHERE clause with UPDATE and DELETE. Without it you change or delete every single row. Before running either, first run a SELECT with the same WHERE to confirm exactly which rows will be affected.

CommandWhat it doesRisk
INSERTAdds new rowsLow — only adds
UPDATEChanges existing rowsMedium — always use WHERE
DELETERemoves rows permanentlyHigh — always use WHERE
